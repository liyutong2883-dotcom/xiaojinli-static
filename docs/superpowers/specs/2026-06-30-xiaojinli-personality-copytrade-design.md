# 小金鲤投资 DNA 抄作业改版设计

日期：2026-06-30

## 背景

当前原型仍然更像传统金融 App：用户进入后先看到搜索、交易热点、热门主题、工具箱和作业列表。这个结构的问题是信息很多，但主线不够锋利。用户来到“小金鲤”的核心动机不是研究市场，而是降低决策成本：我这种赚钱风格该看谁的作业、这个作业为什么赚钱、我能不能抄。

本次改版选择方案 B：重构信息架构。目标不是简单替换首页模块，而是把产品主线改成“趣味投资 DNA 入口 + 个性化作业流 + AI 拆作业”。

## 产品定位

小金鲤不是行情工具，也不是投研终端。它更像一个低门槛、内容化、陪伴式的投资作业发现产品。

核心承诺：

> 先知道你的投资 DNA，再给你看更适合你抄的作业。

产品应该给用户的感觉是“好玩、能看懂、有人带我拆”，而不是“专业、复杂、需要自己研究半天”。

## 目标

1. 把第一触点从行情信息改成五道题趣味测试。
2. 用动物投资 DNA 降低用户理解成本，形成可传播、可记忆的人设标签。
3. 首页优先展示“今天谁赚了大钱”和“适合你的作业”，弱化传统行情热点。
4. 作业详情直接给结论、持仓组合和 AI 解释，减少层层点击。
5. 保留小金鲤 AI 作为随时提问入口。
6. 将工具箱从主流程移出，后续作为独立运营工具处理。

## 非目标

1. 本阶段不做真实交易、开户、下单或资产连接。
2. 本阶段不做真实风控适当性测评，只做趣味投资 DNA 分类。
3. 本阶段不接真实行情或持仓接口，仍使用静态模拟数据。
4. 本阶段不建设完整社区，只预留“灵感/记录”入口。
5. 本阶段不把工具箱作为主导航重点。

## 核心用户流

```text
用户着陆
  -> 五道题趣味测试
  -> 投资 DNA 画像
  -> 个性化作业首页
  -> 看作业：谁赚了大钱
  -> 拆作业：AI 解释持仓逻辑
  -> 抄作业 / 收藏 / 写灵感
  -> 不确定时问小金鲤
```

首次进入时，用户应优先进入测试流程。已有画像用户可以直接进入首页，但首页顶部仍要展示当前投资 DNA 和推荐理由。

## 测试定位

五道题测试命名为“投资 DNA 测试”，不叫心理测试，也不叫风险测评。

原因：

1. 真实目标不是给用户贴心理标签，而是快速建立后续推荐所需的投资画像。
2. 小白用户不一定理解止损、估值、龙头、指数、价值投资等词，第一屏不能让用户有考试感。
3. 题目应该像生活化情景小游戏，而不是传统金融问卷。
4. 每个选项都要能映射到稳定的投资特征，并直接驱动作业、Agent、内容和功能推荐。

测试设计原则：

1. 用户觉得有趣，不像在填专业表格。
2. 用户回答生活里的自己，后台映射投资画像。
3. 后台能提取风险、收益预期、行动速度、信息依赖、情绪稳定性等标签。
4. 测试结果能直接解释“为什么首页给你推荐这些作业”。

## 动物投资 DNA 设计

测试结果前台不叫“人格”，而叫“你的投资 DNA”或“你的赚钱风格”。动物只是记忆钩子，用于降低理解成本，不等同于严肃风险测评。

| 投资 DNA | 前台标题 | 核心倾向 | 推荐作业 | 页面语气 |
| --- | --- | --- | --- | --- |
| 猫头鹰 | 认真研究型 | 先理解原因，再做决定 | AI 解释充分、逻辑清晰的作业 | 理性、解释、强调为什么 |
| 猎豹 | 行动派 | 看到机会愿意快速行动 | 高弹性、趋势、热点作业 | 直接、兴奋、强调机会 |
| 乌龟 | 稳稳赚钱派 | 不喜欢大起大落，更相信时间 | 低回撤、高股息、长期配置作业 | 安心、耐心、强调拿得住 |
| 狐狸 | 消息雷达 | 喜欢先弄清楚发生了什么 | 龙虎榜、事件催化、资讯驱动作业 | 灵活、敏锐、强调变化 |
| 松鼠 | 准备型 | 一点一点积累，对未知保持好奇 | 入门、学习、轻仓模拟作业 | 友好、鼓励、强调从小开始 |

五道题只需要得到一个主 DNA，不需要复杂分数展示。结果页可以展示一个次级倾向，例如“乌龟，带一点猫头鹰”。

## 页面设计

### 1. `onboarding.html` 五道题测试

目的：让用户快速进入产品语境。

结构：

1. 顶部品牌：小金鲤 + “找到你的投资 DNA”。
2. 副标题：只要 30 秒，没有标准答案。
3. 进度条：第 1/5 题。
4. 题目卡片：每题 4 个选项。
5. 底部按钮：下一题 / 查看结果。

题目内容：

1. 如果路边有人排了很长的队，你会？
   - A. 一定有好东西，我也想看看。
   - B. 先看看大家为什么排队。
   - C. 等人少一点再决定。
   - D. 别人排他们的，我有自己的选择。
2. 如果朋友说：“这个东西以后一定会涨价。”你第一反应是？
   - A. 真的吗？我现在就去看看。
   - B. 为什么会涨？
   - C. 我想再等等。
   - D. 网上有没有更多信息？
3. 如果你攒了一笔钱，你更愿意？
   - A. 找机会赚得更多。
   - B. 稳稳增值就好。
   - C. 先放着，不着急。
   - D. 一部分稳一点，一部分试试看。
4. 如果你买的东西第二天就降价了，你会？
   - A. 再买一点，更划算。
   - B. 有点难受，但还能接受。
   - C. 以后买东西要再等等。
   - D. 先看看为什么降价。
5. 如果未来一年，你最希望投资带来什么？
   - A. 赚到一笔让我开心的钱。
   - B. 比放银行更好一点就行。
   - C. 希望自己学会投资。
   - D. 少踩坑，比赚钱更重要。

题目口径：

1. 首版固定使用上面五道题，不出现止损、估值、龙头、指数、价值投资等专业词。
2. 文案像生活选择题，不像金融问卷。
3. 未来可维护一组年轻化题库，例如餐厅排队、红包、新 App、打折、年度目标，但不影响首版主流程。

后台采集标签：

| 标签 | 来源 |
| --- | --- |
| 是否容易 FOMO | Q1、Q2 |
| 是否跟风 | Q1、Q2 |
| 是否独立决策 | Q1、Q2 |
| 决策速度 | Q2、Q3 |
| 风险偏好 | Q2、Q3、Q4、Q5 |
| 信息依赖程度 | Q1、Q2、Q4 |
| 收益预期 | Q3、Q5 |
| 资产配置意识 | Q3 |
| 亏损承受能力 | Q4 |
| 情绪稳定性 | Q4 |
| 是否容易补仓 | Q4 |
| 用户目标 | Q5 |
| 是否成长型用户 | Q5 |
| 是否收益型用户 | Q5 |
| 是否风险规避型用户 | Q5 |

选项映射建议：

| 问题 | A | B | C | D |
| --- | --- | --- | --- | --- |
| Q1 排队 | 热点敏感 / 猎豹 | 信息驱动 / 狐狸 | 稳健等待 / 乌龟 | 独立判断 / 猫头鹰 |
| Q2 涨价推荐 | 快速行动 / 猎豹 | 追问原因 / 猫头鹰 | 延迟决策 / 乌龟 | 查更多信息 / 狐狸 |
| Q3 攒钱选择 | 收益进取 / 猎豹 | 稳健增值 / 乌龟 | 观望准备 / 松鼠 | 分散试探 / 猫头鹰 |
| Q4 买后降价 | 补仓型 / 猎豹 | 情绪稳定 / 乌龟 | 谨慎复盘 / 松鼠 | 原因分析 / 猫头鹰 |
| Q5 投资目标 | 收益期待 / 猎豹 | 稳健期待 / 乌龟 | 学习成长 / 松鼠 | 避坑优先 / 猫头鹰 |

结果可以展示星级雷达：

```text
风险：★★★★☆
行动：★★★☆☆
信息：★★★★☆
稳健：★★☆☆☆
纪律：★★★★☆
```

投资 DNA 映射：

1. 喜欢先问为什么、研究原因、理性决策：猫头鹰。
2. 决策速度快、机会敏感、收益预期高：猎豹。
3. 稳健、低频、重视少踩坑：乌龟。
4. 信息依赖高、喜欢先看发生了什么：狐狸。
5. 偏谨慎、偏学习、对未知保持好奇：松鼠。

交互规则：

1. 每题选择后才能进入下一题。
2. 本静态版可用前端 JS 在本地计算投资 DNA。
3. 结果可写入 `localStorage`，用于首页展示默认投资 DNA。

### 2. `dna_result.html` 投资 DNA 结果

目的：把测试结果转成推荐理由。页面标题应使用“你的投资 DNA”或“你的赚钱风格”，不直接说“你是某某型投资者”。

结构：

1. 大结果卡：动物名称、赚钱风格标题、三句解释。
2. 推荐作业类型：适合看什么、不适合看什么。
3. 风险提醒：一句话提醒，不做长篇教育。
4. 主按钮：去看适合我的作业。

示例：

```text
你的投资 DNA：乌龟
稳稳赚钱派
你更适合先看低回撤、分散配置、能长期拿得住的作业。
```

### 3. `discover.html` 新首页

目的：首页从“发现资讯”改成“看作业”。

首屏结构：

1. 顶部：小金鲤 + 当前投资 DNA 标签。
2. 个性化提示：例如“你的投资 DNA 偏乌龟，今天优先看稳健赚钱榜”。
3. 主模块：今天谁赚了大钱。
4. 三张作业卡：
   - 适合你的作业
   - 今日大赚作业
   - 低回撤赚钱作业

作业卡展示字段：

1. 作业名称 / 人物或机构。
2. 收益率。
3. 最大回撤。
4. 样本数或披露滞后。
5. 投资 DNA 匹配理由。
6. AI 一句话解释。

弱化内容：

1. 交易热点不再作为首屏模块。
2. 热门主题可以变成“更多灵感”入口。
3. 投资运势/日签删除或降级为投资 DNA 氛围小组件。
4. 工具箱不出现在首页主流程。

### 4. `copytrade.html` 作业池

目的：承接首页点击和完整浏览需求。

保留三类作业：

1. 大牛持仓。
2. 游资动向。
3. 私募持仓。

调整方向：

1. 顶部不再强调字段说明，而是强调筛选和排序。
2. 默认排序可以按当前投资 DNA 变化：
   - 猎豹：收益率和趋势强度优先。
   - 乌龟：低回撤优先。
   - 猫头鹰：AI 解释完整度和逻辑清晰度优先。
   - 狐狸：事件催化和热度优先。
   - 松鼠：低门槛、轻仓模拟、学习友好优先。
3. 卡片继续保留成功率、最大回撤、样本数、披露滞后。
4. 每张卡片增加“为什么推荐给你”的短句。

### 5. `portfolio.html` 作业详情 / AI 拆作业

目的：用户点开后直接理解这份作业。

首屏结构：

1. 作业名称。
2. AI 结论：一句话说明这份作业的核心逻辑。
3. 关键指标：收益率、最大回撤、持仓数量、主要风格。
4. 组合持仓：直接展示核心持仓和比例。
5. AI 拆解：为什么这样配。

要删除：

1. “抄作业前先问自己 3 个问题”不作为首屏。
2. 短期 / 中期 / 长期筛选不出现。

风险提示位置：

放在 AI 拆解后面，使用“这份作业不适合谁”的表达，而不是教育用户先反思。

### 6. `inspiration.html` 灵感页

目的：承接“找灵感/写灵感”。

结构：

1. 我的收藏作业。
2. 我的灵感笔记。
3. 小金鲤生成的观察问题。

静态版可先做轻量页面，不需要真实编辑器。支持点击“写下想法”弹出简单输入区域即可。

### 7. `ai_assistant.html` 小金鲤

目的：用户不确定时随时问。

入口：

1. 首页右下角浮球。
2. 作业详情里的“问小金鲤”按钮。
3. 灵感页里的“帮我整理这条想法”按钮。

问题建议：

1. 这份作业适合我吗？
2. 这个组合为什么这么配？
3. 它最大的风险是什么？
4. 如果我的投资 DNA 偏乌龟，我该怎么看它？

### 8. `toolbox.html` 工具箱

处理方式：从主导航移除。

保留文件，但不在底部导航展示。后续如需运营，可以单独做成小程序、外部活动页或二级入口。

## 导航设计

当前：

```text
发现 / 抄作业 / 工具箱 / 我的
```

改为：

```text
首页 / 作业 / 灵感 / 我的
```

说明：

1. 首页：个性化作业流。
2. 作业：完整作业池。
3. 灵感：收藏、记录、写想法。
4. 我的：投资 DNA 画像、关注、设置。

小金鲤 AI 不占底部导航，使用右下角浮球。

## 数据模型

静态版可用前端 JS 对象模拟数据。

作业对象建议字段：

```js
{
  id: "portfolio-turtle-001",
  name: "睿远均衡",
  category: "bull",
  dnaFit: ["turtle", "owl"],
  returnRate: "+18.6%",
  successRate: "63%",
  maxDrawdown: "-18.4%",
  riskLevel: "中低波动",
  sampleSize: "42次调仓",
  disclosureLag: "季报滞后45天",
  aiSummary: "用分散持仓降低波动，用消费和制造维持长期弹性。",
  fitReason: "适合想稳一点、但又不想完全放弃收益弹性的用户。",
  holdings: [
    { name: "宁德时代", weight: "9.5%", reason: "新能源核心仓位" }
  ]
}
```

投资 DNA 对象建议字段：

```js
{
  id: "turtle",
  name: "乌龟",
  headline: "先拿得住，再谈赚多少",
  preferredSort: "drawdown",
  recommendedCategories: ["bull", "private"],
  avoid: "高频短线和高回撤作业"
}
```

## 交互规则

1. 首次进入优先进入测试页。
2. 已有 `localStorage.investmentDna` 时进入首页。
3. 首页内容根据投资 DNA 展示不同推荐标题和排序。
4. 作业详情必须先给 AI 结论，再给持仓细节。
5. 点击“抄作业”在静态版中只模拟收藏/关注状态，不做真实交易。
6. 小金鲤 AI 使用模拟回复，围绕当前作业或投资 DNA 生成解释。

## 测试与验证

新增或调整测试应覆盖：

1. 新页面存在：`onboarding.html`、`dna_result.html`、`inspiration.html`。
2. 底部导航不再包含工具箱。
3. 首页包含“今天谁赚了大钱”。
4. 首页弱化或移除“交易热点”首屏展示。
5. 作业详情包含 AI 结论。
6. 作业详情不包含“抄作业前先问自己 3 个问题”。
7. 作业详情不包含“短期 / 中期 / 长期”筛选。
8. 作业池仍保留三类作业切换。
9. 小金鲤入口仍可访问。

## 实施边界

本次改版建议分三步执行：

1. 信息架构改造：新增测试/画像/灵感页，调整底部导航。
2. 首页重做：从发现页改为个性化作业首页。
3. 作业详情重做：先结论、再持仓、再 AI 拆解。

每一步都应保持静态页面可打开、导航可跳转、测试通过。

## 待确认事项

1. 是否强制首次进入测试页，还是首页顶部提示用户去测试。
2. 五种投资 DNA 是否固定为猫头鹰、猎豹、乌龟、狐狸、松鼠。
3. 首页默认展示收益率榜，还是根据投资 DNA 默认展示不同榜。
4. “灵感”页是偏笔记，还是偏收藏夹。

当前建议：

1. 首次进入强制测试，已有画像后直达首页。
2. 先固定五种投资 DNA。
3. 首页默认展示“适合你的 DNA 榜”，同时提供“今日大赚榜”。
4. 灵感页第一版偏收藏夹 + 轻笔记。
